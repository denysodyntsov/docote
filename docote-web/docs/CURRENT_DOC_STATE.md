# Current Documentation State

## Purpose
The web-first DoCoTe flow should be able to compare code-change context against existing documentation.

## Current step
Add a lightweight in-memory current-document text store and route.

## Why this matters
This creates a first product concept of:
- existing docs in
- impact reasoning out

## Current limitation
Prototype-only, not persistent, not user-scoped.

## Next step
Later connect this to Confluence or uploaded docs.
