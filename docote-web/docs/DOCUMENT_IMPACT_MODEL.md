# Document Impact Model

## Purpose
The core promise of DoCoTe is not only to summarize code changes, but to show how they affect documentation.

## Current step
Add a mock document-impact model that turns likely impacted areas into richer UI items with:
- area
- reason
- severity

## Why this matters
This moves DoCoTe closer to a product that highlights documentation drift and update priority, rather than only generating generic text.

## Next step
Surface document-impact items directly in the analysis UI and later tie them to changed files and real diff metadata.
