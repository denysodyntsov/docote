# Documentation Source Plan

## Purpose
DoCoTe’s web-first track should evolve beyond pasted text and eventually support richer documentation sources.

## Current step
Add a documentation-source concept to the prototype UI.

## Source directions
- manual text
- Confluence
- Word documents

## Why this matters
This reflects the broader product idea that DoCoTe should compare change context against different documentation systems, not only one manual text box.

## Current limitation
The source selector is conceptual only; no real integration is implemented yet.
