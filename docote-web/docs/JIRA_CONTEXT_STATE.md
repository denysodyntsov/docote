# Jira Context State

## Purpose
The web-first DoCoTe track should be able to ingest ticket context even before real Jira integration is implemented.

## Current step
Add a lightweight in-memory Jira context store and route.

## Why this matters
This strengthens the product flow of:
- code/repo context
- current docs context
- Jira context
- merged analysis output

## Current limitation
Prototype-only and not persistent.
