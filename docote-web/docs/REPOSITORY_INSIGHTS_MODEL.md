# Repository Insights Model

## Purpose
Over time, DoCoTe should be able to understand repositories not only as raw repo names, but as meaningful product or service surfaces.

## Current step
Add a lightweight repository-insights layer that derives basic categorization from mocked repo data.

## Why this matters
This opens the door for smarter future behaviors like:
- different documentation suggestions by repo type
- different prompts by platform/client/service repo
- better default expectations for impacted docs
