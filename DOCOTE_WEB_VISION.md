# DoCoTe Web Vision

## Core shift
DoCoTe evolves from a Jira-first documentation helper into a web-first documentation intelligence platform.

## Product thesis
Software teams struggle because delivery knowledge is fragmented across:
- repositories
- pull requests
- commit history
- Jira tickets
- Confluence pages
- ad-hoc documentation

DoCoTe should help teams understand how code changes affect documentation, release communication, and shared delivery knowledge.

## Core promise
Connect your repos, tickets, and docs to keep documentation aligned with every meaningful change.

## Web-first value
A web product allows:
- easier onboarding
- broader integration surface
- support beyond Jira-only teams
- centralised history, diffs, and analysis
- stronger SaaS business model

## Initial user journey
1. Sign in to DoCoTe
2. Connect GitHub or GitLab
3. Select repository
4. Select branch, PR, or commit range
5. Optionally connect Jira / Confluence later
6. Generate:
   - technical summary
   - release summary
   - documentation update draft
   - likely impacted documentation areas
7. Review diffs and track updates over time

## Positioning
DoCoTe is not just a document generator.
It is documentation intelligence for software delivery.

## Preserved earlier work
The Jira-first app remains valuable as:
- a future integration surface
- an internal issue-panel workflow
- an enterprise extension of the core web platform
